## API 导航

从当前任意开发者文档页面进入，并使用其 API 侧边栏。当前文档分类包括事件、窗口、复制、输入、系统、屏幕、用户、本地数据库、动态指令、模拟按键、用户付费、ubrowser、MCP、AI、Sharp 和 FFmpeg。

已知示例：

- 事件：https://www.u-tools.cn/docs/developer/utools-api/events.html
- 本地数据库：https://www.u-tools.cn/docs/developer/utools-api/db.html

不要根据猜测的路径片段拼接未经核验的 API URL。通过官方侧边栏或官方域名搜索，随后引用实际解析出的页面。

## 查阅规则

1. 阅读准确的 API/配置页面，不要只看侧边栏摘要。
2. 在长页面中搜索 API 或字段名称，并检查其所在标题、参数、返回值、注意事项和示例。
3. 当行为依赖其他能力时，继续检查相关链接页面。
4. 打包或提交应用市场前，重新核验发布与 preload 规则。
5. 报告结论时包含访问日期。
